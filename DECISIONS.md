## Конвертация и доставка PDF

**Self-hosted Gotenberg вместо PDFCrowd API.**
PDFCrowd — наследие форкнутого расширения: платный внешний сервис + утёкший API-ключ.
Gotenberg (headless Chromium, HTML→PDF) на своём VPS — без оплаты за вызов, полный контроль.
Эндпоинт пока `http://89.167.13.19:3000` в `background.js` (Stage 5: HTTPS + домен).

**Картинки конвертируются в base64 ДО отправки в Gotenberg.**
Картинки ChatGPT за авторизацией (куки) и lazy-loaded. Gotenberg — отдельный сервер, кук не имеет.
Поэтому инлайним: canvas (same-origin) ИЛИ фоновая докачка с куками (cross-origin / CORS-tainted, как DALL-E).

## Харвест виртуализированного чата

**Скролл-харвест перед клонированием DOM.**
ChatGPT виртуализирует ленту: ответы вне экрана размонтируются (innerHTML пустеет).
Чтобы экспортировать ВЕСЬ чат — прокручиваем сверху вниз, форсируя рендер каждого turn, и кэшируем HTML.

**Харвест учитывает `href` (+ проход на полноту).**
ChatGPT дозагружает `href` ссылок-цитат асинхронно: ранний снимок = синие некликабельные ссылки.
Оцениваем снимки по числу резолвнутых ссылок; довизитим turns с неразрешёнными ссылками.
Фиксированный скролл вдобавок пропускает turns → проход на полноту перевизитит незахваченные.

## DALL-E / картинки в селект-режиме (большая серия багов, 2026-06-26)

**`extractDalleImages` — общая функция для обоих путей экспорта.**
DALL-E картинка — `position:absolute` внутри aspect-box; в PDF коробка схлопывается.
Надо вытащить `<img>` в нормальный поток. Нужно и обычному, и блок-экспорту → вынесли в общую (`render.js`).

**Снапшот ответа восстанавливается ТОЛЬКО для размонтированных turns.**
В блок-режиме снимок ответа берётся в момент клика по чекбоксу — со СТАРЫМ src картинки.
Всегда перетирать клон снимком = затереть base64-конверсию → DALL-E ломается.
Поэтому: размонтированные (пустые) — из снимка; смонтированные — живой клон (уже сконвертированный).

**Картинка опознаётся 3 способами: контейнер `imagegen` + URL + base64.**
Экспорт мутирует живой `<img>` в base64. Детектор только по URL переставал её узнавать ПОСЛЕ
первого экспорта → чекбокс пропадал. Три признака делают детект устойчивым к смене src.

**Селективный экспорт: чистим тело kept-turn от ВСЕГО невыбранного, не только bid'd-блоков.**
Turn остаётся целиком, если в нём есть хоть один выбранный блок; bid-проход удаляет только
элементы С bid. Невыбранные остатки БЕЗ bid (`<hr>`-разделители; каркас невыбранной таблицы/
списка, чьи дети были bid'd) уцелевали и рисовались стопкой пустых горизонтальных линий
(«полоски») ровно там, где было невыбранное содержимое. Поэтому в каждом kept-turn проходим
по детям `.markdown` и удаляем всё, что не является выбранным блоком (или его частью) — ДО
bid-прохода. Так скелеты любого типа не уцелевают. (`blockmode.js`, перед bid-sweep.)

## UX / прочее

**Rate Us — со 2-го успешного экспорта, не с 1-го.** Первый экспорт без трения; меньше навязчивости.
Ключ хранилища `pcr_rated` оставлен как back-compat: хранилище переживает автообновление CWS,
иначе уже оценившие будут спрошены снова.

**Дружелюбные ошибки.** Gotenberg отдаёт сырой `--api-timeout` (503) — юзер с этим ничего не сделает.
Переводим в «чат слишком большой — экспортируй меньше сообщений».

**Единый префикс `gptpdf`.** Было два легаси-префикса (`pdfcrowd`, `pcr`) от форка PDFCrowd.
Свели к одному осмысленному. Коллизию `single-page` развели:
`gptpdf-single-page` (id кнопки / класс экспорта) ≠ `gptpdf-singlepage-toggle` (тумблер настроек).

**Настройки — во встроенной модалке, без отдельной options-страницы.**
Standalone options page — артефакт форка, удалён. Настройки = модалка + 2 быстрых тумблера + ratebar.

## Сквозное правило

**Не переписывать с нуля.** Запутанный код кодирует hard-won edge cases ChatGPT-DOM:
виртуализация, двойной KaTeX, sr-only лейблы, DALL-E-обёртки, CORS-фавиконки. Рефакторить слоями.

## HTTPS / прод-инфраструктура (2026-06-26)

**Caddy перед Gotenberg (reverse proxy + TLS-терминация).**
Gotenberg — только HTTP и без авторизации; выставлять его голым в интернет нельзя. Ставим Caddy:
он держит сертификат, расшифровывает на 443 и проксирует на `localhost:3000`. Caddy выбран за
авто-HTTPS — сам получает и продлевает сертификат Let's Encrypt (вызов был через TLS-ALPN-01).

**Домен через DuckDNS (бесплатный поддомен `export-gpt.duckdns.org`), не платный.**
Сертификат выдают только на ИМЯ, не на голый IP — поэтому домен обязателен. Для приватности данных
платный vs бесплатный не различаются (DNS трафик не видит, только имя→IP). Платный — вопрос солидности,
прикручивается позже заменой одной строки в Caddyfile.

**Внешний порт 3000 закрыт; `localhost:3000` остаётся.**
После переезда на https напрямую к Gotenberg снаружи ходить незачем — закрыли inbound 3000 в
Hetzner-фаерволе (плюс к безопасности). Caddy→Gotenberg по localhost: петлевой трафик фаервол не
фильтрует, потому ничего не сломалось.

**Опубликованная версия — на PDFCrowd, не на Gotenberg.**
Важно для миграции: текущие пользователи в сторе НЕ ходят на наш VPS (старый код = PDFCrowd API).
CWS сам обновит их на новую (Gotenberg+HTTPS) версию. Поэтому закрытие 3000 живых пользователей не
задело, «переходного периода» на сервере нет. (И поэтому утёкший PDFCrowd-ключ всё ещё актуально отозвать.)
